1. DDL
CREATE : 테이블 생성
 	- CREATE TABLE [NAME] ( [R_NAME] [R_TYPE], .... );
 	- CREATE TABLE [NAME] AS SELECT [R_NAME] FROM [NAME];

ALTER : 테이블 구조 변경 
	- ALTER TABLE [NAME] ADD [R_NAME] [R_TYPE] ; # 열 추가
    	- ALTER TABLE [NAME] RENAME COLUMN [OLD_NAME] TO [NEW_NAME]; # 열 이름변경
    	- ALTER TABLE [NAME] MODIFY [R_NAME] [NEW_TYPE]; # 열 자료형 변경
	- ALTER TABLE [NAME] DROP COLUMN [R_NAME]; # 열 삭제

RENAME : 테이블 이름 변경
	- RENAME TABLE [OLD_NAME] TO [NEW_NAME]; 

TRUNCATE : 테이블 데이터 삭제
	- TRUNCATE TABLE [NAME]; 
	
DROP : 테이블 삭제
    	- DROP TABLE [NAME];  


2. DML
SELECT : 데이터 조회
	- SELECT [R_NAME] FROM [NAME] WHERE [조건];
	- SELECT * FROM [NAME]; # 전체 조회

INSERT : 데이터 추가
	- INSERT INTO [NAME] (R_NAME, ...) VALUES (VALUE, ...);
	- INSERT INTO [NAME] VALUES (VALUE1, ...); # 모든 행에 데이터 추가
	- INSERT INTO [NAME] SELECT [R_NAME] FROM [NAME];
	
UPDATE : 데이터 수정
	- UPDATE [NAME] SET [R_NAME] = [VALUE] WHERE [조건];

DELETE : 데이터 삭제
	- DELETE {NAME} FROM [NAME] WHERE [조건];
 
* TRUNCATE는 트랜잭션 로그를 생성하지 않고, 롤백이 불가능
  DELETE는 테이블 내의 행(row)을 하나씩 삭제하는 작업이기 때문에 데이터를 삭제할 때마다 트랜잭션 로그를 생성하고, 롤백(rollback)이 가능

3. DCL
GRANT : 권한 부여
	- GRANT [객체권한] ON [스키마,객체이름..] TO [사용자이름,롤이름..];
REVOKE : 권한 부여 취소
	- REVOKE [객체권한] ON [스키마,객체이름..] TO [사용자이름,롤이름..];


4. TCL
COMMIT : 트랜잭션에서 실행한 모든 작업을 영구적으로 DB에 반영
ROLLBACK : 트랜잭션이 시작된 이후에 발생한 변경사항을 원래 상태로 되돌림
SAVEPOINT : 트랜잭션 내에서 부분적으로 롤백할 수 있는 지점을 설정
