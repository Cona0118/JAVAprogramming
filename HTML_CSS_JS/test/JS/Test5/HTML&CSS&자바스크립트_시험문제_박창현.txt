1. 상품 주문 페이지 구현

아래는 상품 주문 페이지를 구현하는 과제입니다. 
제공된 CSS를 활용하여 레이아웃을 구성하고, JavaScript를 사용하여 동적인 기능을 구현하시오.
------------------------------------------------------------------------------------------------------
<파일 구조>
 - order.html : 상품 주문 화면
 - address_popup.html : 주소 검색 팝업
 - order_complete.html : 주문 완료 화면
------------------------------------------------------------------------------------------------------
<구현되어야 할 기능>
1-1.상품 주문 화면(order.html)
상품명과 가격은 변경되지 않음. (기본값 적용 ->무선 이어폰/59900)
수량 입력 가능 (기본값: 1개)
수량 변경 시 총 금액 자동 계산(
주소 입력칸 + "주소검색" 버튼 → 팝업창을 통해 주소 입력 가능
결제 수단 선택 (신용카드, 계좌이체, 간편결제 중 1개만 선택)
이메일 입력 필드 (형식 검증 필요)
배송 요청사항 입력 가능(선택 사항)
약관 동의 체크박스 (동의 필수)
주문하기 버튼 클릭 시 입력값 검증 후 주문 완료 페이지로 이동

1-2.주소 검색 팝업(address_popup.html)
주소 입력 필드와 확인 버튼
확인 버튼 클릭 시 입력한 주소를 부모창(order.html)으로 전달하고 팝업 닫기
새로고침 시에, 팝업이 여러번 호출되지 않도록 구현

1-3.주문 완료 화면(order_complete.html)으로 페이지 이동
"주문이 완료되었습니다!" 메시지 출력
------------------------------------------------------------------------------------------------------
<구현 조건 및 요구사항>

- 수량 검증 : 1개 이상이어야 하며, 0 이하일 경우 주문 불가
- 주소 입력 : 팝업창을 통해 입력된 주소가 자동으로 반영(부모창의 주소입력 텍스트는 readonly적용)
- 결제 수단 : 3가지 중 하나는 반드시 선택
- 이메일 유효성 검증 : @와 .이 포함된 올바른 이메일 형식이어야 함(정규표현식 사용하지 않음)
- 약관 동의 필수 : 체크하지 않으면 주문 불가
- 총 금액 실시간 반영 : 수량 변경 시 가격 × 수량 자동 계산 (원화표현시, toLocaleString() 사용)
                             (수량 변경 시 input 이벤트 사용 -> 값이 바뀔 때마다 즉시 실행) 
- 팝업 창 기능 : 부모창과 데이터 연동 (window.opener 사용)
- 페이지 이동 방식 : 주문 완료 시 order_complete.html로 이동
